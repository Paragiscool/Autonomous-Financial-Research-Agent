Data Quality Errors: 
●​ Stale data: Using cached financial data from a previous quarter when current data is available. 
●​ Conflicting sources: Different data sources report different figures for the same metric (for 
example, revenue figures from a financial API may not match those in the 10-K due to 
restatements). 
●​ Misattribution: The agent correctly retrieves data but attributes it to the wrong company or 
time period. 
●​ Unit confusion: Mixing millions and billions, confusing percentages with basis points, or 
misinterpreting currency denominations. 
 
A4.3 Error Handling Strategies 
Retry With Exponential Backoff: 
For transient errors (API timeouts, rate limits, temporary unavailability), the agent should implement 
retry logic with exponential backoff. The initial retry delay should be 1 second, doubling with each 
subsequent attempt up to a maximum of 5 retries. Each retry should include a jitter component 
(random delay of 0-500ms) to prevent thundering herd problems when multiple agent instances hit the 
same API. 
 
Fallback Tool Chains: 
For each primary tool, define one or more fallback tools that can provide similar (if less precise) 
information. The fallback chain for financial data might be: (1) primary financial data API, (2) SEC 
filing extraction with parsing, (3) web search for financial summary sites, (4) vector database for 
previously stored data. The agent should transparently log when it uses a fallback and note the 
reduced confidence in its output. 
 
Fact Verification Pipeline: 
Every numerical claim in the agent's output should pass through a fact verification step. The 
verification pipeline works as follows: (1) extract all numerical claims from the draft report, (2) for 
each claim, identify the source tool and raw data, (3) cross-reference against at least one independent 
source, (4) flag any claim where sources disagree by more than 5% for manual review, (5) attach 
confidence scores to each claim in the final report.

Graceful Degradation Protocol: 
When errors cannot be resolved, the agent must degrade gracefully rather than fail silently. Graceful 
degradation means: (1) clearly stating in the report which sections could not be completed and why, 
(2) providing the best available information from tools that did work, (3) suggesting alternative 
approaches the user could take to obtain the missing information, (4) never fabricating data to fill 
gaps left by failed tool calls.

SECTION A5: EVALUATION FRAMEWORK DESIGN 
A5.1 Why Evaluation Is Hard For Research Agents 
The Evaluation Challenge: 
Evaluating an autonomous research agent is fundamentally harder than evaluating a classification 
model or a recommendation system. Research quality is multi-dimensional: a report can be factually 
accurate but poorly structured, comprehensive but incoherent, well-written but missing critical risks. 
Furthermore, there is rarely a single 'correct' research output for a given query. Two expert analysts 
researching the same company may produce substantially different reports, both of high quality, 
because they prioritize different aspects of the analysis. 
 
A5.2 Quality Metrics Framework (20+ Metrics) 
Category 1: Factual Accuracy (5 Metrics): 
●​ FA-1: Numerical Accuracy Rate – Percentage of numerical claims in the report that match 
authoritative sources. Target: >98%. 
●​ FA-2: Citation Accuracy – Percentage of cited sources that are real, accessible, and actually 
support the claims attributed to them. Target: 100%. 
●​ FA-3: Temporal Accuracy – Whether the report correctly identifies the time periods for all 
data points (e.g., distinguishing Q3 2024 data from Q3 2023). Target: 100%. 
●​ FA-4: Entity Accuracy – Whether all company names, executive names, product names, and 
ticker symbols are correct. Target: 100%. 
●​ FA-5: Hallucination Rate – Number of claims in the report that cannot be traced to any source 
retrieved by the agent. Target: 0. 
 
Category 2: Completeness (4 Metrics): 
●​ CO-1: Section Coverage – Whether the report includes all sections specified in the template 
(executive summary, financial analysis, risk assessment, competitive positioning, outlook). 
Target: 100%. 
●​ CO-2: Data Source Diversity – Number of distinct source types used (SEC filings, earnings 
calls, news, financial data, industry reports). Target: >=4 source types. 
●​ CO-3: Temporal Coverage – Whether the report covers historical trends (3-5 year horizon) in 
addition to current data. Target: >=3 years. 
●​ CO-4: Risk Factor Coverage – Percentage of material risk factors identified compared to the 
company's own 10-K risk factor disclosures. Target: >=80%.
