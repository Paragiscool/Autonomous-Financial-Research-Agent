AI And Agent Terminology 
Agentic AI: A software system that autonomously perceives its environment, reasons about goals, 
selects and executes actions through tool invocation, and adapts its behaviour based on feedback. 
Distinguished from chatbots by its ability to take multi-step actions without human guidance at each 
step. 
 
ReAct (Reasoning + Acting): An agent architecture pattern where the LLM alternates between 
generating reasoning traces (Thought) and executing tool calls (Action), processing tool outputs 
(Observation) before continuing. Introduced by Yao et al. (2022) in the paper 'ReAct: Synergizing 
Reasoning and Acting in Language Models.' 
 
Plan-and-Execute: An agent architecture pattern where a Planner LLM generates a complete 
research plan before any tools are invoked. An Executor then carries out the plan step by step, with 
optional plan revision when intermediate results require changes. 
 
Tool Registry: The central catalog of all tools available to the agent, including their names, 
descriptions, input/output schemas, authentication requirements, rate limits, and fallback alternatives. 
The registry is injected into the LLM's system prompt to enable tool selection. 
 
Context Window: The maximum number of tokens (words/sub-words) that an LLM can process in a 
single request. GPT-4o supports 128K tokens; Claude Sonnet 4 supports 200K tokens. Context 
window management is critical for research agents that accumulate large amounts of retrieved data. 
 
Hallucination: When an LLM generates content that is factually incorrect, fabricated, or unsupported 
by any retrieved source. In financial research, hallucinations are particularly dangerous because they 
may involve plausible-sounding but entirely incorrect financial figures. 
 
RAG (Retrieval Augmented Generation): An architectural pattern where the LLM's generation is 
grounded in externally retrieved documents rather than relying solely on its training data. The

Vector Database: A database optimized for storing and searching high-dimensional vector 
representations (embeddings) of text. Used for semantic search where queries are matched by 
meaning rather than exact keyword match. Key options include Pinecone, Weaviate, Chroma, and 
Qdrant. 
 
Embedding: A dense numerical vector representation of text, generated by an embedding model. 
Text with similar meanings produces vectors that are close together in the embedding space (high 
cosine similarity). Used by vector databases for semantic search. 
 
Chunking: The process of splitting long documents into smaller segments (chunks) before 
embedding and storing in a vector database. Financial documents require semantic chunking that 
respects document structure rather than naive fixed-size splitting. 
 
System Prompt: The initial instructions provided to the LLM that define its role, capabilities, 
constraints, and output format. In an agent, the system prompt includes the full tool registry and 
behavioural guidelines. System prompt design is a critical skill for agent architects. 
 
Chain-of-Verification: A prompt pattern where the LLM generates a draft response, then 
re-examines that draft to identify claims requiring verification, retrieves verification evidence, and 
produces a corrected final response. 
 
Graceful Degradation: The ability of a system to continue operating with reduced functionality 
when components fail, rather than crashing entirely. In a research agent, this means producing partial 
but transparent reports when some tools are unavailable. 
 
Circuit Breaker: A software pattern that detects cascading failures and stops routing requests to a 
failed component after a threshold number of failures. Prevents a single tool failure from consuming 
all the agent's retry budget and blocking other operations. 
LLM-as-Judge: An evaluation technique where an LLM is used to assess the quality of another 
LLM's output against a rubric. Used for qualitative metrics like analytical depth and narrative quality 
that are difficult to measure with automated metrics alone.

Financial Terminology 
 
10-K Filing: The annual report filed by publicly traded companies with the SEC. Contains audited 
financial statements, management discussion and analysis (MD&A), risk factor disclosures, executive 
compensation, and other material information. The most authoritative source of company information. 
 
10-Q Filing: The quarterly report filed with the SEC. Contains unaudited financial statements and 
updated MD&A. Less comprehensive than the 10-K but more current. 
 
8-K Filing: A current report filed with the SEC to announce material events such as leadership 
changes, mergers and acquisitions, bankruptcy filings, or other developments that shareholders should 
know about promptly. 
 
DEF 14A (Proxy Statement): Filed before the annual shareholder meeting, this document discloses 
executive compensation, board composition, shareholder proposals, and other governance matters. 
 
Earnings Call Transcript: A written record of the quarterly conference call where company 
management presents financial results and answers questions from analysts. Contains both prepared 
remarks and a Q&A section that often reveals management's forward-looking views. 
 
EBITDA: Earnings Before Interest, Taxes, Depreciation, and Amortization. A commonly used 
measure of a company's operating performance that removes the effects of financing and accounting 
decisions. EBITDA Margin = EBITDA / Revenue. 
 
Free Cash Flow (FCF): Cash generated by the business after capital expenditures. Calculated as 
Operating Cash Flow minus Capital Expenditures. A key measure of a company's ability to generate 
cash for shareholders. 
P/E Ratio (Price-to-Earnings): The ratio of a company's stock price to its earnings per share. Used 
to assess whether a stock is overvalued or undervalued relative to its earnings. Can be calculated 
using trailing twelve months (TTM) or forward estimated earnings.
