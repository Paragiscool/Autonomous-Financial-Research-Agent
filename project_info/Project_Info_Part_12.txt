SECTION E1: CORE FRAMEWORK – LANGCHAIN / LANGGRAPH 
E1.1 LangChain Overview 
LangChain is the most widely adopted open-source framework for building LLM-powered 
applications. It provides abstractions for prompt management, tool integration, chain composition, 
and agent creation. For this project, LangChain provides the foundational building blocks for your 
agent's tool integration layer and chain-of-thought execution. 
 
Key LangChain Components You Will Use: 
●​ ChatOpenAI / ChatAnthropic: LLM wrapper classes that handle API communication, retry 
logic, and response parsing. 
●​ Tool: The base class for defining agent tools with name, description, and function references. 
●​ AgentExecutor: The runtime that manages the ReAct loop, parsing LLM outputs into tool calls 
and feeding observations back. 
●​ PromptTemplate: Template system for constructing system prompts with variable injection 
(tool descriptions, conversation history). 
●​ OutputParser: Parses structured LLM outputs into Python objects for downstream processing. 
 
E1.2 Langgraph Overview 
LangGraph extends LangChain with stateful, graph-based workflows. While LangChain's 
AgentExecutor runs a simple loop, LangGraph allows you to define complex workflows as directed 
graphs where nodes represent processing steps and edges represent conditional transitions. This is 
essential for implementing the Plan-and-Execute pattern and for building multi-agent systems where 
different agents handle different aspects of the research. 
 
Key Langgraph Concepts: 
●​ StateGraph: Defines the agent's workflow as a graph with a shared state object. 
●​ Nodes: Each node is a function that reads the current state, performs processing (LLM call, 
tool execution, data transformation), and updates the state. 
●​ Conditional Edges: Transitions between nodes can be conditional, allowing the agent to take 
different paths based on intermediate results.
